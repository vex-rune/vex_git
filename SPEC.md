# GitVex 功能规格文档

> 对标 GitHub Desktop，适配手机端交互，纯 Dart 实现（git_on_dart）

---

## 一、基础规范

### 配色规范
| 操作 | 色彩 |
|------|------|
| 拉取更新 | 绿色 |
| 推送同步 | 蓝色 |
| 新增文件 | 绿色 |
| 修改文件 | 橙色 |
| 删除文件 | 红色 |
| 重命名 | 蓝色 |
| 复制 | 紫色 |

### 页面布局
- 左上角：汉堡菜单按钮
- 中间主体：默认文件预览浏览页
- 右上角：本地变更文件数量角标（无变动隐藏）

---

## 二、汉堡菜单完整功能

### 1. 仓库项目板块
- **仓库列表**：点击条目一键切换仓库，切换后刷新预览页面
- **新建仓库**：本地空白仓库创建
- **克隆仓库**：填入远程地址下载仓库
- **导入本地仓库**：扫描手机目录绑定已有 Git 仓库
- **仓库管理**：重命名、移除仓库、修改本地存储路径

### 2. 密钥凭证管理板块
#### HTTPS 令牌管理
- 查看已绑定账号、PAT 密钥
- 新增、编辑、删除访问凭证

#### SSH 密钥管理
- 查看本机公私钥信息
- 一键生成密钥、导入外部密钥
- 在线校验密钥连通有效性

#### 凭证绑定
- 可单独为单个仓库配置专属访问密钥

### 3. 系统设置入口
- 定时巡检周期设置：10分钟 / 20分钟 / 自定义
- 外观样式、通知提醒开关
- 清除缓存、恢复默认配置

### 4. 分支快捷入口
- 快速调出分支列表
- 执行切换、新建、删除分支操作

---

## 三、首页预览页核心逻辑

### 1. 本地变更实时监测
- 后台持续扫描文件，识别新增、修改、删除、重命名
- 右上角实时刷新变更数量，例：3 个文件改动显示数字 3
- 点击数字角标，直接跳转提交编辑页面
- 提交完毕、同步完成后，数字归零隐藏

### 2. 定时远程版本巡检
- 按设定周期静默比对本地与远程仓库版本节点
- 仅校验差异，不自动下载文件、不改动本地数据
- 校验结果分支：
  - 版本完全一致：无提示，等待下一轮巡检
  - 远端存在新版本：弹出绿色提醒「检测到 X 个文件可更新」

### 3. 基础浏览操作
- 点击文件夹展开层级，点击文件直接查看内容
- 支持上下滑动浏览目录，缩放查看文本内容

---

## 四、提交编辑页面闭环逻辑

1. 页面展示全部变更文件，标注新增/修改/删除类型
2. 支持单个勾选、全选批量勾选待提交文件
3. 选中文件可弹窗查看行级代码差异
4. 填写提交标题、补充备注说明
5. 点击提交按钮生成本地提交记录
6. 提交成功自动返回首页预览页，变更数量清零
7. 支持撤回本次未确认提交操作

---

## 五、拉取更新流程（绿色标识）

1. 收到远端更新提示，手动点击绿色拉取按钮
2. 前置校验本地状态：
   - 存在未提交改动：弹窗提供「暂存修改」「放弃修改」「取消拉取」三个选项
   - 无未提交内容：正常拉取远端最新数据
3. 拉取过程显示进度状态
4. 拉取完成自动刷新文件预览页面
5. 若出现文件冲突，自动跳转冲突处理页面

---

## 六、推送同步流程（蓝色标识）

1. 本地提交完成后，点击蓝色推送按钮
2. 自动比对远程分支最新版本
3. 状态判定：
   - 本地远程版本同步：直接推送提交记录至云端
   - 远程已有新提交：弹窗提示版本不一致，禁止直接推送
4. 存在文件冲突时，进入可视化编辑页处理差异
5. 冲突解决完毕，再次发起推送即可同步成功

---

## 七、冲突处理页面逻辑

1. 自动标记冲突文件与冲突代码段落
2. 三种处理选项：保留本地内容、保留远程内容、手动合并编辑
3. 编辑完成后生成合并提交记录
4. 处理结束回到原页面，可继续执行拉取或推送操作

---

## 八、分支操作完整逻辑

1. 从汉堡菜单进入分支面板
2. 查看全部本地分支、远程追踪分支
3. 切换分支：
   - 切换前检测未提交文件，提示留存改动或丢弃改动
   - 确认后切换分支，刷新首页文件预览
4. 新建分支：基于当前分支衍生新分支，名校验合规性
5. 删除分支：限制主干分支删除权限，普通分支可正常移除

---

## 九、账号密钥异常闭环处理

1. 拉取、推送、克隆时校验凭证有效性
2. 密钥过期、权限不足、账号失效时，弹出报错提醒
3. 弹窗快捷跳转至密钥管理页面，重新编辑配置凭证
4. 重新保存密钥后，可再次重试本次同步操作

---

## 十、通用状态与提示规则

| 状态 | 表现 |
|------|------|
| 操作中 | 展示加载动画、进度条 |
| 操作成功 | 简短文字弹窗提示结果 |
| 操作失败 | 区分网络错误、密钥错误、版本冲突、文件损坏分类提示 |

- 所有弹窗均配备取消、确认按钮，操作可中途终止

## 十三、已知限制（git_on_dart 0.1.4）

| 功能 | 状态 | 说明 |
|------|------|------|
| clone | ✅ | 支持 HTTPS/SSH |
| pull/push | ✅ | 需要 origin remote 已配置 |
| commit/log | ✅ | 完整支持 |
| branch/checkout | ✅ | 完整支持 |
| merge | ✅ | 支持，有冲突处理 |
| status | ✅ | staged/unstaged/untracked |
| diff | ⚠️ | 行级 diff 暂不支持 |
| unstage | ❌ | 暂不支持 reset HEAD |
| SSH 密钥生成 | ❌ | 需外部生成 |
| 行级代码对比 | ❌ | 暂不支持 |
| 定时巡检 | ⏳ | 需自实现后台任务 |
